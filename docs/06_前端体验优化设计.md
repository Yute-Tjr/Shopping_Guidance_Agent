# 06 · 前端体验优化设计（Phase 6）

> 本文档用于 Phase 6 前端 App 界面优化规划。范围限定在 iOS 原生 SwiftUI 客户端，不改 Phase 1-5 的后端 RAG、图搜、ASR/TTS 主链路。目标是把现有可用界面打磨成克制、可信、像真实电商导购工具的体验，而不是“AI 应用模板”。

---

## 1. 背景与目标

Phase 5 已完成文本、多轮、图搜、ASR 语音输入、TTS 播报和音色选择。当前前端的主要问题不是功能缺失，而是视觉语言还偏“暖橙 + 圆角卡片 + 装饰化品牌感”，容易显得像 demo。Phase 6 的重点是：

1. **降低 AI 感**：不使用发光背景、渐变光斑、过度漂浮阴影、泛用 sparkles 装饰。
2. **减少圆角卡片堆叠**：普通容器圆角控制在 8pt 内；只有头像、图标按钮、输入胶囊允许更圆。
3. **提高信息密度**：推荐商品、价格、规格、对比信息要更像购物决策工具，而不是宣传卡片。
4. **统一交互状态**：上传、录音、识别、生成、准备朗读、缓存朗读、错误降级都有明确但克制的反馈。
5. **动效服务状态变化**：只做短、轻、可解释的转场；关闭 Reduce Motion 时仍保持完整可用。

## 3. 设计方向

**方向名：Calm Commerce Desk**

界面应该像一个安静的购物决策台：清晰、可靠、可比较。品牌可以有记忆点，但不抢商品和回复内容的注意力。

### 3.1 视觉原则

- **背景**：使用近白 / 轻灰底，不使用发光、渐变光斑、bokeh、装饰 orb。
- **色彩**：主色仍保留 PriceCat 橙，但从大面积填充改为小面积强调；价格红和省钱绿只用于价格语义。
- **圆角**：`card=8`、`chip=6`、`bubble=10~12`、`hero=8`。图标按钮可保持圆形。
- **阴影**：默认取消大阴影，改为 1px 边线 + 轻分隔。只有可点击商品项允许极轻阴影。
- **字体**：中文正文优先系统 sans，提高可读性；品牌 wordmark 或价格可保留少量 serif。避免整页 serif 导致信息界面像海报。
- **图标**：保留 SF Symbols，但减少 sparkles、cat 装饰的使用频率；功能按钮优先用语义图标。

### 3.2 不做清单

- 不做全屏 hero、营销落地页式欢迎屏。
- 不做发光背景、紫蓝渐变、玻璃拟态堆叠。
- 不把每个区域都包成圆角大卡片。
- 不用夸张弹跳动画；不让动画改变列表布局稳定性。
- 不在 App 内写“你可以这样使用本功能”的长说明。

## 4. 信息架构与界面安排

### 4.1 Chat 主界面

**顶部栏**

- 左侧：PriceCat 小型 wordmark + 一行状态短语。
- 中间不放大标题，不占用垂直空间。
- 右侧：音色菜单、自动播报开关、新建会话。
- 新建会话按钮使用 `plus.bubble` 或 `square.and.pencil`，不再用高饱和圆形背景。

**空状态**

- 第一屏直接显示可操作的输入区和 4 个 starter prompt。
- starter prompt 按语义分组：`美妆`、`数码`、`对比`、`拍照找货`。
- 每个 prompt 是一行列表项，左侧小图标，右侧箭头；圆角 8，弱边框，不加阴影。
- 猫图标缩小为品牌符号，不做 96pt 大图形中心装饰。

**消息列表**

- 保留 `List`，继续利用 UITableView 高度缓存，避免流式 token 卡顿。
- Assistant 文本气泡宽度按内容 hug，但最大宽度控制在 86%。
- 用户消息不使用大面积亮橙；改为浅橙底 + 左/右侧 3pt 品牌色竖条，提升长文本可读性。
- 表格、商品列表不嵌在气泡内部，作为独立内容块。

**输入栏**

- 底部输入栏为固定 dock：顶部一条 1px 分隔线，背景与 canvas 区分 2-3 个灰阶。
- 左侧工具：图片、麦克风。
- 中间文本输入：圆角不超过 14，focus 时仅边线变深。
- 右侧发送：可发送时实心品牌色图标按钮；不可发送时灰化。
- 选图后显示一条紧凑 image shelf：缩略图 44x44 + 文件状态 + 删除按钮。
- 上传、ASR、TTS 准备状态共用同一种 inline status row，避免多种提示样式。

### 4.2 Assistant 回复与商品推荐

**回复正文**

- 流式阶段使用轻量纯文本 + 小型呼吸点。
- 完成后再 markdown 解析，表格独立全宽。
- 播报按钮放在回复块右下角，命中缓存时不显示准备提示；未缓存时在输入栏上方显示“正在准备朗读...”。

**商品推荐列表**

- 商品推荐从“卡片”改为“商品行”：左图、右信息、底部决策理由。
- 主图固定 96x96，圆角 6。
- 右侧信息顺序：品牌 / 类目、标题、价格、规格数量。
- reason 不再做 chip，改为灰底细线提示或纯文本二级信息，最多 2 行。
- “省 ¥xx” badge 只在价格差明确时显示，位置靠近价格，不压在图片上。
- 点击反馈：轻微 scale 到 0.985，120ms 回弹，不改变布局。

### 4.3 商品详情页

**页面结构**

1. 顶部返回导航 + 简短标题。
2. 主图 4:3，占满宽度，圆角 8，不使用大阴影。
3. 商品基础信息：品牌、类目、标题、价格区间。
4. 决策信息区：推荐理由、适用场景、限制条件。
5. SKU 表格：规格属性左对齐，价格右对齐，行间用分隔线而非卡片堆叠。
6. 底部保留“返回对话”或“复制商品 ID”级别的轻操作，不新增未实现的购物车主 CTA。

**状态**

- loading：主图骨架 + 三行文本骨架，避免只有一个 spinner。
- error：显示错误原因 + 重试按钮。
- 图片加载失败：保留固定比例占位，不让布局跳动。

### 4.4 图片输入流程

**选图前**

- 图片按钮常驻输入栏左侧。
- 图标使用 `photo.on.rectangle`，不写“拍照找货”长文字。

**选图后**

- image shelf 显示缩略图、状态文案、删除按钮。
- 文案只写业务状态：`待发送`、`上传中`、`已按文字继续`。
- 发送后用户气泡顶部展示缩略图，宽度随图片比例但最大 180。

**失败**

- 503 降级提示为一行：`图片检索暂不可用，已按文字继续`。
- 不展示技术错误如 embedding、Pillow、fallback flag。

### 4.5 语音输入与 TTS

**语音输入**

- 麦克风按钮点击后进入录音态：按钮变实心，输入栏上方出现低高度 waveform。
- waveform 高度 12-16pt，使用 5-7 根竖条，不发光。
- 停止后识别中显示 inline status：`正在识别...`。
- ASR 空文本时不展示错误；保持输入框为空。

**TTS**

- Header 音色菜单显示当前音色的短名称，例如 `灿灿`。
- 自动播报开关只用 icon 状态变化，不额外弹提示。
- 手动播报：未缓存显示准备提示；缓存命中直接播放。
- 播放中按钮可切换为 `stop.circle`，点击停止当前播报。

## 5. 动画细节

所有动效默认遵守 `Reduce Motion`。开启 Reduce Motion 时保留 opacity 变化，取消位移、缩放和波形循环。

| 场景 | 动效 | 参数 |
| --- | --- | --- |
| 消息插入 | opacity 0→1 + y 8→0 | 160ms easeOut |
| assistant 流式点 | opacity 0.35↔1 | 700ms easeInOut repeat |
| 商品行点击 | scale 1→0.985→1 | 120ms + 100ms |
| image shelf 出现 | opacity + y 6→0 | 140ms easeOut |
| 录音开始 | 麦克风按钮 fill 切换 + waveform fade in | 140ms |
| 录音 waveform | 竖条 scaleY 随音量变化 | 80ms linear，限制刷新频率 |
| TTS 准备提示 | status row fade in/out | 120ms |
| 新建会话 | 列表 opacity out，空状态 opacity in | 180ms，不做大位移 |
| 商品详情进入 | 使用系统 Navigation push | 不自定义夸张转场 |

实现要求：

- 用 `.animation(_:value:)` 绑定明确状态，不给大容器挂无约束 implicit animation。
- 优先动画 `opacity`、`scaleEffect`、`offset`，避免动画 `frame`、`padding`、`List` 行高。
- 列表滚动仍保持当前 80ms 节流策略，不能为了动画恢复每 token 滚动动画。

## 7. 验收标准

### 7.1 视觉验收

- 第一屏不能像营销 landing page；必须直接可对话。
- 屏幕中不能出现大面积发光、光斑、渐变背景。
- 主界面不应被多层圆角卡片包裹。
- 商品信息在 390pt 宽度下标题、价格、规格不重叠。
- 中文长标题、长品牌、长 reason 均能截断或换行，不溢出。

### 7.2 交互验收

- 新建会话后输入框、history、录音、播报、上传图状态全部重置。
- 图 + 文、纯文、语音回填、TTS 播报四条链路都可用。
- TTS 首次未缓存显示准备提示；缓存命中直接播放，不显示准备提示。
- ASR 空文本不显示内部错误。
- 用户滚动历史时，流式 token 不强行把列表拉到底。

### 7.3 工程验收

- `git diff --check` 通过。
- iOS Xcode build 必须通过。
- 若 SwiftPM 测试环境可用，跑 `swift test`；如本机 `import Testing` 环境不可用，必须说明原因并以 Xcode build 为构建门槛。
- 手动验收覆盖：推荐蓝牙耳机、对比两款商品、图片找同款、语音输入 + TTS 播报。

## 8. 开发注意事项

- 不重写 ViewModel 主流程；Phase 6 是界面打磨，不碰 RAG 逻辑。
- 不引入第三方 UI / 动画库。
- 不使用 WebView 或跨平台框架。
- 保留当前 `List` 实现，避免回到 ScrollView + LazyVStack 的流式性能问题。
- 每个固定格式 UI 元素要有稳定尺寸：工具按钮、商品图、状态行、SKU 行。
- 动效必须可取消、可降级，不制造输入延迟。

# 06 · 打磨与交付开发文档

> 本文档记录 Phase 6 的交付态实现：iOS 原生 SwiftUI 前端体验精修、开场动画、ECS Docker 沙箱部署、Nginx 公网入口、静态商品图公网化、以及 iOS 真机 HTTP 沙箱联调。Phase 6 不改变 RAG / 图搜 / ASR / TTS 的核心业务链路，重点是把已有能力整理成可演示、可部署、可复跑的交付形态。

---

## 1. 本轮目标

本轮基于已接入的 `PriceCat_icon` 品牌图标、Phase 5 语音能力和服务端沙箱部署需求，完成前端体验、服务端部署与联调闭环：

1. **底部输入区统一**：相册、麦克风、文本输入、发送按钮收进同一个 composer 控制组，状态提示和选图条统一放在 composer 上方。
2. **推荐问题卡片分化**：空状态的 4 条 starter prompt 不再共用搜索图标，改为按语义配置图标和强调色。
3. **逐段朗读控制组精修**：每条 assistant 回复下方保留朗读按钮，并把音色选择紧贴到同一个胶囊控制组内；顶部不再放全局音色和朗读按钮。
4. **开场动画**：将 `docs/assets/app_begin.mp4` 复制到 iOS bundle，作为 App 冷启动后的全屏品牌动画。
5. **服务端沙箱部署**：补齐 Dockerfile / Compose / Nginx 运行方式，后端容器只暴露给宿主机本地 `127.0.0.1:8000`，公网入口统一走 Nginx 80/443。
6. **公网联调配置**：iOS 沙箱默认连 ECS 公网入口，服务端通过 `STATIC_BASE_URL` 生成公网商品图 URL；正式发布前切换到 HTTPS 域名。

设计基调延续 PriceCat 图标：浅鼠尾草绿背景、黑猫墨色主操作、猫眼黄绿点缀、少量暖铜色用于错误或礼物类提示。

## 2. 视觉与交互原则

- **减少散点控件**：底部不再是多个孤立按钮，而是一个稳定的输入 dock。
- **语义优先图标**：推荐问题图标服务商品意图，不使用泛用 sparkle 或重复搜索符号。
- **触控面积稳定**：相册、麦克风、发送、朗读按钮保持接近 44pt 的可触达面积。
- **状态贴近输入**：上传、语音、TTS 准备提示都出现在底部输入区上方，不散落在消息流中。
- **朗读按消息归属**：音色选择跟随每条 assistant 回复，用户能明确知道“这段话将用哪个音色播报”。
- **不过度装饰**：继续避免发光 orb、重渐变、大面积装饰卡片，保留真实工具型 App 的克制感。

## 4. Starter Prompt 设计

默认推荐项从纯字符串升级为 `StarterPrompt`：

| 场景 | 文案 | 图标 | 色彩角色 |
| --- | --- | --- | --- |
| 护肤 | 推荐一款适合油皮的洗面奶 | `drop.fill` | `skincare` / 信号绿 |
| 数码 | 200 元以下的蓝牙耳机 | `headphones` | `audio` / 炭灰 |
| 对比 | 对比一下兰蔻和雅诗兰黛的精华 | `rectangle.2.swap` | `compare` / 猫眼黄绿 |
| 礼物 | 送女朋友的口红选什么色号 | `gift.fill` | `gift` / 暖铜色 |

这样后续如果要接入“近期热搜”或个性化推荐，只需要替换数据源，不需要在 `ChatView` 里继续写死图标和颜色。

## 5. 底部 Composer

新的输入区分为三层：

1. **状态层**：上传降级、语音错误、TTS 准备态共用 `inputStatusRow`，统一图标、字号、边框和背景。
2. **图片层**：选图后显示 `pickedImageShelf`，固定 44x44 缩略图，右侧提供移除按钮。
3. **Composer 层**：相册、麦克风、文本框、发送按钮在同一个圆角容器内，focus 时只加深外框，不额外弹出装饰。

输入区背景使用 `Theme.Palette.surfaceTint` 与页面 canvas 拉开层级，顶部保留 1pt 分隔线，避免底部操作区和消息列表粘在一起。

## 6. 逐段朗读与音色选择

assistant 回复完成后，如果文本非空且可播报，则在该回复下方显示朗读控制组：

- 左侧：黑猫墨色圆形朗读按钮，使用 `speaker.wave.2.fill`。
- 中间：细分隔线，表达朗读动作和音色选择属于同组但不是同一动作。
- 右侧：当前音色名称 + `waveform` + `chevron.down`，点击后展示 `SpeechVoice.all`。
- 选中态：菜单内对当前音色显示 `checkmark`。

这个结构替代顶部全局音色/朗读按钮，避免用户在长对话里不清楚当前设置会影响哪段回复。

## 7. 开场动画

开场动画使用 `AVPlayerLayer` 直接渲染 `app_begin.mp4`，不使用 `VideoPlayer`，避免出现系统播放控件。视频在主界面上方以 full-bleed overlay 展示，主界面仍在底层初始化，动画结束后只做 220ms opacity 淡出。

交互与降级规则：

- 视频默认静音播放，避免 App 冷启动时突然出声。
- 右上角提供“跳过”按钮，满足长视频或重复启动时的退出需求。
- `AVPlayerItemDidPlayToEndTime` 后自动进入主界面。
- 播放失败时直接进入主界面，不让用户卡在启动页。
- 当前冷启动策略是资源存在就播放视频；`LaunchIntroState` 保留 Reduce Motion 策略分支，后续如需严格遵守无障碍偏好可接入运行时环境值。
- `docs/assets/app_begin.mp4` 只作为设计素材存档，真正打包资源是 `client/ShoppingGuide/Resources/app_begin.mp4`。

## 8. 服务端沙箱部署

### 8.1 部署拓扑

当前沙箱部署在阿里云 ECS 上，入口链路如下：

```
公网用户 / iOS App
  │
  ▼
http://121.196.247.225:80
  │
  ▼
Nginx
  │ proxy_pass http://127.0.0.1:8000
  ▼
Docker Compose 沙箱
  ├─ pricecat_api    FastAPI / Uvicorn / /api/v1 / /static
  └─ pricecat_mysql  MySQL 8
```

安全组只需要开放：

| 端口 | 用途 | 公网开放 |
| --- | --- | --- |
| 22 | SSH | 仅限可信 IP |
| 80 | HTTP 沙箱入口 | 是 |
| 443 | HTTPS 预留 | 是 |
| 8000 | FastAPI 宿主机回环端口 | 否 |
| 3306 | MySQL 容器内部访问 | 否 |

### 8.2 Docker 约束

`server/Dockerfile` 做了三件部署态处理：

1. **阿里云镜像源**：`apt` 和 `pip` 都切到阿里云镜像，避免 ECS 访问 Debian / PyPI 海外源构建过慢。
2. **数据集双路径兼容**：离线脚本默认读取 `/ecommerce_agent_dataset`；FastAPI 静态挂载从 `/app/ecommerce_agent_dataset` 查找。镜像内保留数据集原路径，并创建 `/app/ecommerce_agent_dataset -> /ecommerce_agent_dataset` 软链，保证灌库、建索引、`/static/...` 商品图都能访问同一份数据。
3. **持久化数据卷**：Milvus Lite 和上传缓存放在宿主机 `server/data`，通过 compose 挂载到容器 `/app/data`，避免容器重建后索引文件丢失。

`docker-compose.sandbox.yml` 中 API 端口写成：

```yaml
ports:
  - "127.0.0.1:8000:8000"
```

因此外部不能直接访问 `:8000`，公网访问必须经过 Nginx。

### 8.3 环境变量

服务器上的 `server/.env` 需要比本地多一个公网静态资源地址：

```env
MYSQL_DSN=mysql+asyncmy://shopping_user:shopping_pwd@mysql:3306/shopping_guide?charset=utf8mb4
MILVUS_DB_PATH=/app/data/milvus_lite.db
STATIC_BASE_URL=http://121.196.247.225
```

`STATIC_BASE_URL` 只在服务器真实 `.env` 中填写，仓库里的 `server/.env.example` 保持空值：

```env
STATIC_BASE_URL=
```

这样别人 clone 仓库本地运行时仍会回退到：

```text
http://127.0.0.1:8000/static/...
```

不会被强制连到当前 ECS。

### 8.4 复跑命令

ECS 上更新部署：

```bash
cd /opt/pricecat-sandbox/price_cat
git pull

docker compose -f docker-compose.sandbox.yml build api
docker compose -f docker-compose.sandbox.yml up -d --force-recreate api
```

首次部署或数据集变化后初始化：

```bash
docker compose -f docker-compose.sandbox.yml exec api python -m app.db.init_db
docker compose -f docker-compose.sandbox.yml exec api python -m scripts.seed_mysql --truncate
docker compose -f docker-compose.sandbox.yml exec api python -m scripts.build_index --rebuild
docker compose -f docker-compose.sandbox.yml exec api python -m scripts.build_image_index --rebuild
```

只修改 `.env`、`STATIC_BASE_URL` 或 Nginx 时，不需要重新灌库和建索引，只需要重启 API 容器：

```bash
docker compose -f docker-compose.sandbox.yml up -d --force-recreate api
```

## 9. 公网联调与图片链路

### 9.1 iOS BaseURL

沙箱期 `AppEnvironment.baseURL` 指向：

```swift
URL(string: "http://121.196.247.225")!
```

不要写 `:8000`，因为公网入口是 Nginx 的 80 端口。发布前应改为从配置注入：

```text
Debug:   http://127.0.0.1:8000 或沙箱 IP
Release: https://api.your-domain.com
```

### 9.2 ATS 临时例外

由于当前沙箱使用 `http://公网IP`，iOS 端显式使用 `client/ShoppingGuide-Info.plist` 并临时允许 HTTP：

```xml
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsArbitraryLoads</key>
    <true/>
</dict>
```

这是沙箱联调策略，不是发布态策略。提交 App Store 前必须移除该例外，并改成 HTTPS 域名。

### 9.3 商品图片 URL

后端卡片 `image_url` 由：

```text
STATIC_BASE_URL + /static/ + image_path
```

拼出。沙箱期应返回类似：

```text
http://121.196.247.225/static/1_美妆护肤/images/p_beauty_001_live.jpg
```

如果终端里看到 JSON 转义形态：

```text
1_\u7f8e\u5986\u62a4\u80a4/...
```

这是 JSON 展示，不应把反斜杠转义串原样粘到浏览器地址栏。验证静态资源时可用浏览器打开实际中文路径，或使用百分号编码 URL。

验收命令：

```bash
curl -s http://121.196.247.225/api/v1/products/p_beauty_001 | python3 -m json.tool | grep image_url
curl -I "http://121.196.247.225/static/1_%E7%BE%8E%E5%A6%86%E6%8A%A4%E8%82%A4/images/p_beauty_001_live.jpg"
```

## 10. 验收标准

### 视觉验收

- 空状态 4 条推荐项拥有不同图标和轻量强调色。
- 底部输入区看起来是一个完整 composer，而不是独立按钮拼接。
- 选图、语音/TTS 状态提示不会挤压输入框，也不会遮挡消息列表。
- assistant 回复下方朗读控件保持轻量，不喧宾夺主。
- 冷启动后开场动画全屏显示，右上角“跳过”不遮挡主体画面。

### 交互验收

- 点击推荐项会把对应文案写入输入框并发送。
- 相册、麦克风、文本输入、发送按钮仍保持原有行为。
- 选图后可移除，发送后沿用既有图片上传逻辑。
- 每条 assistant 回复旁边可直接播报，并可在同一位置切换音色。
- 开场动画播放结束、播放失败或点击跳过后，都能进入聊天主界面。
- 真机 App 使用 `http://121.196.247.225` 可访问聊天、图搜、语音接口；不再访问本地 `127.0.0.1:8000`。
- 商品卡片图片 URL 指向 `STATIC_BASE_URL/static/...`，真机和浏览器都能打开。

### 工程验收

- 新增 `StarterPromptTests` 锁定 prompt 元数据，防止后续误把 4 条推荐项退回同一图标。
- 新增 `LaunchIntroStateTests` 锁定启动动画的状态分支。
- `swift test` 用于验证逻辑层和 prompt 元数据。
- `xcodebuild` 用于验证 SwiftUI 视图层、资源目录和 iOS App 编译。
- `docker compose -f docker-compose.sandbox.yml config --quiet` 用于验证沙箱编排文件。
- `docker compose -f docker-compose.sandbox.yml ps` 中 `pricecat_api` / `pricecat_mysql` 均应为 `Up`。
- `curl http://127.0.0.1:8000/healthz` 与 `curl http://121.196.247.225/healthz` 均应返回 `{"app":"ok","db":"ok"}`。
- 如需视觉回归，使用模拟器启动 App 后截图检查开场动画、空状态、底部 composer 和消息朗读控制组。

## 11. 发布前收口

- 配置正式域名并通过 Nginx / Certbot 切 HTTPS。
- 将 iOS `baseURL` 从硬编码公网 IP 改为 Debug / Release 配置注入。
- 移除 `NSAllowsArbitraryLoads`，确保 Release 只访问 HTTPS。
- 将 `STATIC_BASE_URL` 从 `http://121.196.247.225` 改为 `https://api.your-domain.com`。
- 安全组继续保持 `8000` / `3306` 不对公网开放。
- `.env`、API Key、证书私钥不进入 Git；仓库只保留 `.env.example`。

## 12. 后续可选优化

- 给 composer 增加轻量录音波形，但必须遵守 Reduce Motion。
- 给消息朗读按钮增加播放中 stop 状态，和 `SpeechSpeaking.stop()` 对齐。
- 商品卡片后续可继续从“营销卡片”收敛为更高信息密度的商品行。
- 如果接入真实个性化 starter prompt，保留 `StarterPrompt` 作为 UI 数据边界。
